# Product Analysis

## Workflow

1. Clarify the user problem and desired behavior.
2. Check whether the feature belongs in MVP 1, MVP 2, or MVP 3.
3. Identify user stories, success criteria, and non-goals.
4. Identify health, privacy, and AI safety constraints.
5. Define acceptance criteria that can be tested.
6. Hand off to feature planning when the feature is clear enough.

## Rules

- Do not expand MVP scope without naming the trade-off.
- Do not frame the product as medical diagnosis or treatment.
- Prefer structured state and daily behavior loops over chat-only features.
