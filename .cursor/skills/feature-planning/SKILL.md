# Feature Planning

## Workflow

1. Start from the user problem and MVP scope.
2. Identify affected domains, API contracts, screens, and data model changes.
3. Split the feature into vertical slices that can be reviewed independently.
4. Assign work to backend, frontend, design system, and test roles.
5. Define acceptance criteria and validation commands.
6. Call out risks, unknowns, and decisions needed before implementation.

## Output Shape

- Goal
- Scope
- Out of scope
- Backend tasks
- Frontend tasks
- Design system tasks
- Test tasks
- Acceptance criteria
- Risks and open decisions
